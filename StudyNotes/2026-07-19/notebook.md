# 2026-07-19 ACT 项目学习记录: 

## 拿到一个新项目应该怎么阅读

**寻找项目的主线和数据流。**

### 1. 先看 README

README 是项目的入口地图。先通过它了解：

- 这个项目是做什么的
- 有哪些核心脚本
- 如何安装依赖
- 作者推荐的运行命令是什么
- 项目支持哪些任务

对于机器学习项目，README 里的 example usage 特别重要，因为它通常直接暴露了训练、评估、数据生成的主入口。

### 2. 找配置文件

配置文件回答的是项目的“基本设定”，比如：

- 有哪些任务
- 数据集目录在哪里
- 每个 episode 多长
- 使用哪些相机
- 状态和动作维度是多少

在 ACT 项目中，这些信息主要在 `constants.py` 里。

### 3. 理解数据契约

所谓数据契约，就是搞清楚：

- 模型吃什么？
- 模型输出什么？
- 数据从哪里来？
- 每个样本的 shape 是什么？


对于模仿学习项目，尤其要关注：

- observation 是什么
- action 是什么
- episode 是什么
- 数据是单步的，还是一整段轨迹
- 训练时模型预测的是一个动作，还是未来一段动作

### 4. 找数据生成入口

如果项目包含数据生成脚本，应该优先看它。因为它告诉我们数据最原始的样子。

在 ACT 中，数据生成入口是：record\_sim\_episodes.py

它负责让仿真机器人执行 scripted policy，然后把 observation 和 action 保存成 `.hdf5` 文件。

### 5. 找数据读取入口

数据保存之后，还要看训练时怎么读取。

在 ACT 中，数据读取逻辑主要在：utils.py
核心类是：EpisodicDataset
它会从 `.hdf5` 文件中读取：image, qpos, action, is\_pad
然后交给训练循环。

### 6. 找训练入口

训练入口一般负责：
解析命令行参数
- 加载配置
- 加载数据
- 创建模型
- 训练
- 保存 checkpoint

在 ACT 中，训练入口是：imitate\_episodes.py

### 7.拆解多层封装的方法：

**不断追问：这个函数接收什么、返回什么、谁调用它、它又调用谁。**
